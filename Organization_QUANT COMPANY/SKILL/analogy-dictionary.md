# 📖 Quant Analogy Dictionary — พจนานุกรมอุปมาอุปไมย

คู่มือเทียบคำศัพท์ Quant กับภาษาชีวิตจริง จัดกลุ่มตามหมวดหมู่

---

## 📊 หมวด: ตัวชี้วัดผลงาน (Performance Metrics)

| คำศัพท์ | ความหมายจริง | อุปมาอุปไมย |
|---|---|---|
| **Sharpe Ratio** | ผลตอบแทนต่อหน่วยความเสี่ยง | "ความคุ้มค่า — เหมือนถามว่าข้าวร้านนี้อร่อยคุ้มกับที่ต้องรอคิวไหม? ยิ่งสูงยิ่งคุ้ม" |
| **Sortino Ratio** | ผลตอบแทนต่อความเสี่ยงด้านขาดทุน | "เหมือน Sharpe แต่สนใจแค่ 'วันที่ร้านขาดทุน' ไม่สนวันกำไร — ดูว่าเสี่ยงขาดทุนหนักแค่ไหน" |
| **Max Drawdown** | การขาดทุนสูงสุดจากจุดสูงสุด | "หลุมลึกที่สุดที่ตกลงไป — เหมือนลงจากยอดเขาลงหุบเขา ลึกแค่ไหนก่อนจะปีนกลับขึ้นมา" |
| **Win Rate** | สัดส่วนการเทรดที่ชนะ | "ถ้าตีลูก 100 ครั้ง โดนกี่ครั้ง — Win Rate 60% = ตี 10 ครั้ง โดน 6 ครั้ง" |
| **Profit Factor** | กำไรรวม ÷ ขาดทุนรวม | "เหมือนอัตราส่วนรายรับต่อรายจ่าย — ได้ 1.5 หมายความว่าทุก 1 บาทที่เสีย ได้กลับมา 1.5 บาท" |
| **CAGR** | อัตราเติบโตเฉลี่ยต่อปี | "ถ้าร้านคุณโตปีละเท่าไหร่ เฉลี่ยตลอดช่วงที่เปิด" |
| **Recovery Period** | ระยะเวลาฟื้นตัวจาก Drawdown | "ล้มแล้วกี่วันถึงลุกขึ้นยืนได้เท่าเดิม" |
| **Calmar Ratio** | CAGR ÷ Max Drawdown | "กำไรต่อปีเทียบกับหลุมลึกสุด — ร้านกำไรดีแค่ไหนเทียบกับช่วงที่เลวร้ายที่สุด" |
| **Expectancy** | ค่าคาดหวังต่อการเทรด 1 ครั้ง | "โดยเฉลี่ย ทุกครั้งที่เทรด 1 ครั้ง คาดว่าจะได้/เสียเท่าไหร่" |
| **Payoff Ratio** | กำไรเฉลี่ยต่อครั้ง ÷ ขาดทุนเฉลี่ยต่อครั้ง | "ชนะทีได้เท่าไหร่ เทียบกับแพ้ทีเสียเท่าไหร่ — เหมือนเวลาตกปลา ได้ปลาตัวใหญ่ไหมเทียบกับจำนวนครั้งที่เบ็ดขาด" |
| **Volatility** | ความผันผวนของผลตอบแทน | "ถนนขรุขระแค่ไหน — Volatility สูง = ขับรถบนถนนลูกรัง สั่นตลอด" |

## 🔬 หมวด: การวิจัยและพัฒนา (Research & Development)

| คำศัพท์ | ความหมายจริง | อุปมาอุปไมย |
|---|---|---|
| **Alpha** | ผลตอบแทนส่วนเกินจากตลาด | "ความได้เปรียบ — เหมือนมีสูตรลับที่ทำอาหารอร่อยกว่าร้านอื่น" |
| **Alpha Decay** | Alpha ที่ลดลงตามเวลา | "สูตรลับที่คนอื่นเริ่มรู้ จึงหมดความพิเศษ เหมือนอาหารสดที่เน่าเร็ว" |
| **Edge** | ความได้เปรียบทางสถิติ | "ไพ่เหนือกว่า — เหมือนเล่นเกมที่มีกฎเอื้อให้เราชนะบ่อยกว่าแพ้นิดหน่อย" |
| **Hypothesis** | สมมติฐานที่ต้องพิสูจน์ | "คำทำนายที่ต้องพิสูจน์ — เช่น 'ถ้าฝนตกวันจันทร์ คนจะซื้อร่มวันอังคาร' ต้องไปหาข้อมูลมายืนยัน" |
| **Statistical Significance** | ความน่าเชื่อถือทางสถิติ | "ผลที่ได้ไม่ได้เกิดจากโชค — เหมือนโยนเหรียญได้หัว 8 จาก 10 ครั้ง ไม่น่าจะบังเอิญ" |
| **P-value** | ค่าความน่าจะเป็นที่ผลเกิดจากโชค | "โอกาสที่ผลนี้เกิดจากฟลุค — ยิ่งต่ำยิ่งดี เหมือนบอกว่า 'โอกาสที่ร้านนี้แค่โชคดี มีแค่ 2%'" |
| **Regime** | สภาวะตลาดที่แตกต่างกัน | "ฤดูกาล — ตลาดก็มีหน้าร้อน หน้าฝน หน้าหนาว กลยุทธ์ที่ดีในหน้าร้อนอาจไม่ดีในหน้าฝน" |
| **Mean Reversion** | ราคากลับสู่ค่าเฉลี่ย | "ลูกยางเด้งกลับ — ราคาไปไกลเกินไปแล้วมักจะดีดกลับมา" |
| **Momentum** | แนวโน้มต่อเนื่อง | "รถไฟที่วิ่งแล้วหยุดยาก — ราคาที่ขึ้นแรงมักขึ้นต่อสักพัก" |

## 🛡️ หมวด: การจัดการความเสี่ยง (Risk Management)

| คำศัพท์ | ความหมายจริง | อุปมาอุปไมย |
|---|---|---|
| **Position Sizing** | ขนาดการลงทุนต่อครั้ง | "สั่งอาหารทีละจาน — ไม่สั่งทั้งเมนู ถ้าไม่อร่อยก็เสียไม่มาก" |
| **Correlation** | ความสัมพันธ์ระหว่างสินทรัพย์ | "ไข่ในตะกร้า — Correlation สูง = ไข่อยู่ตะกร้าเดียวกัน ตกทีเดียวแตกหมด" |
| **Exposure** | มูลค่ารวมที่เสี่ยงอยู่ | "เงินที่วางบนโต๊ะ — ยิ่งวางมากยิ่งเสี่ยงมาก" |
| **Hedging** | การป้องกันความเสี่ยง | "ทำประกัน — จ่ายนิดหน่อยแต่ถ้าเกิดอะไรขึ้นก็ไม่เจ็บหนัก" |
| **VaR (Value at Risk)** | ขาดทุนสูงสุดที่อาจเกิดขึ้น | "สถานการณ์เลวร้ายที่สุดที่เตรียมรับ — เหมือนถามว่า 'ถ้าวันนี้โชคร้ายสุดๆ จะเสียเงินมากสุดเท่าไหร่?'" |
| **Risk/Reward Ratio** | อัตราส่วนความเสี่ยงต่อผลตอบแทน | "เสี่ยง 1 เพื่อได้ 2 — เหมือนเดิมพัน 100 เพื่อลุ้นชนะ 200" |
| **Circuit Breaker** | กลไกหยุดระบบเมื่อขาดทุนเกินกำหนด | "สะพานไฟ — ถ้าไฟลัดวงจรก็ตัดไฟทั้งบ้านเพื่อป้องกันไฟไหม้" |
| **Diversification** | การกระจายความเสี่ยง | "อย่าใส่ไข่ในตะกร้าใบเดียว — กระจายไปหลายตะกร้า ตกตะกร้าหนึ่งก็ยังเหลืออีก" |
| **Leverage** | การใช้เงินกู้ขยายการลงทุน | "มีดที่คมมาก — ใช้ดีก็หั่นเร็ว ใช้ไม่ดีก็บาดลึก" |

## 🧪 หมวด: การทดสอบและตรวจสอบ (Testing & QA)

| คำศัพท์ | ความหมายจริง | อุปมาอุปไมย |
|---|---|---|
| **Overfitting** | โมเดลจำข้อมูลเก่าเกินไป | "นักเรียนที่ท่องจำเฉลยแทนที่จะเข้าใจ — สอบเก่าได้ 100 แต่เจอโจทย์ใหม่ทำไม่ได้" |
| **Look-ahead Bias** | ใช้ข้อมูลอนาคตในการตัดสินใจ | "ดูเฉลยก่อนสอบ — ผลดีเกินจริงเพราะโกง" |
| **Survivorship Bias** | ดูแค่ตัวที่รอด ไม่ดูตัวที่ตาย | "ถามแต่คนรวยว่าทำยังไง ไม่ถามคนที่ล้มเหลว — ภาพบิดเบือน" |
| **Walk-forward Test** | ทดสอบแบบเลื่อนช่วงเวลาไปเรื่อยๆ | "ซ้อมมวยกับคู่ต่อสู้ใหม่ทุกรอบ — ไม่ใช่ซ้อมกับคนเดิมตลอด" |
| **Out-of-sample** | ข้อมูลที่ไม่เคยเห็น ใช้ทดสอบจริง | "สอบจริง — ไม่ใช่แบบฝึกหัดที่ทำมาแล้ว" |
| **In-sample** | ข้อมูลที่ใช้พัฒนากลยุทธ์ | "แบบฝึกหัด — ข้อมูลที่ใช้เรียนรู้และปรับจูน" |
| **Data Leak** | ข้อมูลรั่วไหลจาก Test เข้า Train | "คำตอบหลุดเข้าไปในห้องสอบ — ผลที่ได้เชื่อถือไม่ได้" |
| **Robustness** | ความทนทานของกลยุทธ์ | "รถที่วิ่งได้ทุกสภาพถนน — ไม่ใช่รถแข่งที่วิ่งได้แค่บนแทร็ก" |
| **Stress Test** | ทดสอบในสถานการณ์สุดขั้ว | "ซ้อมหนีไฟ — จำลองว่าถ้าเกิดเหตุร้ายที่สุดจะรับไหวไหม" |
| **Forward Test** | ทดสอบกับตลาดจริง (ไม่ใช้เงินจริง) | "ฝึกงานก่อนได้ตำแหน่งจริง — ทำทุกอย่างเหมือนจริงแต่ยังไม่มีเงินเดิมพัน" |
| **Paper Trading** | เทรดจำลอง | "เล่นเกมจำลอง — ทำเหมือนจริงทุกอย่างแต่ไม่มีเงินจริง" |
| **Backtest** | ทดสอบย้อนหลังกับข้อมูลเก่า | "ดูว่าถ้าใช้วิธีนี้ในอดีต จะรอดไหม — เหมือนย้อนเวลาไปทดสอบแผนของเรา" |

## ⚙️ หมวด: การดำเนินการ (Execution & Infrastructure)

| คำศัพท์ | ความหมายจริง | อุปมาอุปไมย |
|---|---|---|
| **Slippage** | ราคาจริงต่างจากราคาที่สั่ง | "สั่งข้าวมันไก่ 50 บาท แต่ต้องจ่าย 51 — ส่วนต่างนิดหน่อยที่หลีกเลี่ยงไม่ได้" |
| **Latency** | เวลาหน่วง | "ความเร็วอินเทอร์เน็ต — ยิ่งช้ายิ่งเสียเปรียบ เหมือนกดซื้อตั๋วช้ากว่าคนอื่น" |
| **Fill Rate** | สัดส่วนคำสั่งที่ดำเนินการได้ | "สั่ง 10 อย่าง ได้มากี่อย่าง — 95% Fill Rate = สั่ง 10 ได้ 9.5" |
| **Uptime** | เวลาที่ระบบทำงานปกติ | "ร้านเปิดกี่ชั่วโมง — 99.9% Uptime = ปิดซ่อมแค่ 8 ชั่วโมงใน 1 ปี" |
| **Pipeline** | ระบบลำเลียงข้อมูลอัตโนมัติ | "สายพานในโรงงาน — ส่งของจากจุด A ไป B ไป C อัตโนมัติ" |
| **Deployment** | การนำระบบขึ้นใช้งานจริง | "เปิดร้านจริง — จากที่ซ้อมทำอาหารที่บ้าน ตอนนี้เปิดร้านขายลูกค้าจริงแล้ว" |

## 📐 หมวด: การจัดการพอร์ต (Portfolio Management)

| คำศัพท์ | ความหมายจริง | อุปมาอุปไมย |
|---|---|---|
| **Portfolio Composition** | ส่วนประกอบของพอร์ต | "เมนูอาหาร — มีกี่จาน จานไหนเป็นจานหลัก จานไหนเป็นของหวาน" |
| **Rebalancing** | ปรับสัดส่วนพอร์ต | "จูนเสียงกีตาร์ — ปรับให้กลับมาเท่าเดิมหลังจากเล่นไปสักพัก" |
| **Weight / Allocation** | น้ำหนักหรือสัดส่วน | "แบ่งเค้ก — ชิ้นไหนได้ส่วนมาก ชิ้นไหนได้น้อย" |
| **Drawdown Limit** | ขาดทุนสูงสุดที่ยอมรับได้ | "เส้นตาย — ขาดทุนถึงจุดนี้ต้องหยุดทันที เหมือนตั้งงบประมาณสูงสุด" |
| **Max Exposure** | ความเสี่ยงรวมสูงสุดที่ยอมรับ | "จำนวนลูกบอลสูงสุดที่โยนได้พร้อมกัน — เยอะเกินไปก็หลุดมือ" |

## 📝 วิธีใช้พจนานุกรมนี้

1. **ค้นหาคำ** ที่ต้องการแปลจากตาราง
2. **ใช้อุปมาอุปไมย** เป็นจุดเริ่มต้น แล้วปรับให้เข้ากับบริบท
3. **ประยุกต์** อุปมาให้เข้ากับสถานการณ์จริงของ CEO
4. ถ้าไม่มีในพจนานุกรม ให้ **สร้างอุปมาใหม่** ตามหลักการเดียวกัน

### หลักการสร้างอุปมาใหม่:
- เลือกสิ่งที่ **คนส่วนใหญ่เคยเจอ** ในชีวิตจริง
- เน้น **แก่นของความหมาย** ไม่ใช่รายละเอียดปลีกย่อย
- อุปมาต้อง **ไม่ทำให้เข้าใจผิด** — ถ้ามีข้อจำกัดของอุปมา ให้บอกด้วย
- ทดสอบว่า **คนที่ไม่รู้เรื่อง Quant อ่านแล้วเข้าใจไหม**
